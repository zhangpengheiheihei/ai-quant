一、分类型机器学习算法解析

1.1 什么是"分类"

分类（Classification）是有监督学习中最常见的任务：给定一组带标签样本 (x, y)，其中 y 取自离散集合（二分类时 y ∈ {0, 1}），学习一个函数 f 使得对于新样本能够输出正确的类别或类别概率。二分类是应用最广的形态：垃圾邮件识别、病理良恶性判断、信用违约预测、股票次日涨跌预测——本质都是"是 / 否"问题。

1.2 决策树（Decision Tree）

决策树用"一系列 if-else 判断"来做分类。训练时算法在每个节点上选择信息增益（ID3）、信息增益比（C4.5）或基尼不纯度（CART）最优的特征与阈值作为切分标准，把样本递归划分到两个子节点，直到叶节点足够纯。

  • 优点：结果高度可解释（"若 RSI < 30 且 5 日均线上穿 20 日均线，则判涨"），无需特征标准化，支持类别型与数值型特征混用。
  • 缺点：单棵深树容易过拟合，对样本微小扰动敏感；边界呈阶梯状，无法平滑刻画连续关系。

1.3 逻辑回归（Logistic Regression, LR）

虽名"回归"，实为线性二分类器。先做线性组合 z = w·x + b，再通过 Sigmoid 函数 σ(z) = 1/(1 + e-z) 将结果压缩到 [0, 1] 作为正类概率。训练时最大化对数似然（等价于最小化交叉熵），通常配合 L1/L2 正则防止过拟合。

  • 优点：训练与预测极快，参数具有可读的对数几率解释（每个特征系数 = 变动一个单位使 log-odds 增加多少），是量化、风控中最常见的基线模型。
  • 缺点：只能拟合线性决策边界；对特征间非线性交互不敏感，需人工构造交互项或多项式特征。

1.4 随机森林（Random Forest, RF）

Bagging 思想的经典代表：在训练集上做有放回抽样生成多个子样本，每棵树在每个分裂节点上还从随机的特征子集中挑最优特征，最终由若干棵去相关的决策树投票。

  • 优点：几乎自动抗过拟；能捕获非线性交互；带自然的特征重要性输出；对异常值、缺失值容忍度高。
  • 缺点：模型体积大、推理慢；对训练集范围外的数据外推能力弱；难像单棵树那样人肉解释。

1.5 支持向量机（SVM）

SVM 寻找一个把两类样本分开且间隔最大的超平面；通过核函数（如 RBF、多项式）可以在隐式的高维空间中处理非线性问题。

  • 优点：小样本高维数据表现好；决策边界几何直观。
  • 缺点：核函数、C、γ 参数敏感；训练复杂度约为 O(n²)，大样本很慢；概率输出需额外 Platt Scaling。

1.6 梯度提升树（GBDT）

Boosting 家族：每棵新树拟合前一轮预测的残差（负梯度方向），逐步叠加得到强分类器。XGBoost、LightGBM、CatBoost 都属于这一家族，是 Kaggle 与量化领域的常胜将军。

  • 优点：单模型精度高；较简单的特征工程即可达到不错效果；能给出特征重要性。
  • 缺点：训练串行不好并行；超参较多；样本外调优要小心过拟合。

1.7 算法对比一览

二、模型评价指标解析

2.1 混淆矩阵（Confusion Matrix）

混淆矩阵把预测结果与真实结果按 2×2 方阵组织，是所有二分类指标的"母体"。

预测为正 (1)预测为负 (0)
实际为正 (1)TP 真正例FN 假负例（漏报）
实际为负 (0)FP 假正例（误报）TN 真负例

由此派生的核心指标：

  • 准确率 Accuracy = (TP + TN) / 总数 —— 整体命中率；类别不均衡时会失真。
  • 精确率 Precision = TP / (TP + FP) —— "预测为正的样本里真正的比例"；量化下预测涨的股票里真涨的比例。
  • 召回率 Recall（灵敏度） = TP / (TP + FN) —— "真正的样本被抓到的比例"；医学中的漏诊率互补指标。
  • F1 = 2·P·R / (P + R) —— 精确率与召回率的调和平均，兼顾两者。
  • 特异度 Specificity = TN / (TN + FP) —— 反面指标，衡量正确排除非正样本的能力。

2.2 ROC 曲线（Receiver Operating Characteristic）

ROC 曲线以 FPR = FP/(FP+TN) 为横轴、TPR = TP/(TP+FN) 为纵轴，随着判定阈值从 1 逐步下降到 0，模型在每个阈值下取一个 (FPR, TPR) 点，把这些点连成的曲线就是 ROC 曲线。

  • 左上角 (0, 1)：完美分类器；
  • 对角线 y = x：等价于抛硬币的瞎猜；
  • ROC 曲线越靠左上，模型区分正负类的能力越强。

2.3 AUC（Area Under the ROC Curve）

AUC 是 ROC 曲线下的面积，直观意义是：随机取一个正样本与一个负样本，模型给正样本的分数高于负样本的概率。

  • AUC = 0.5：模型没有区分能力，等于瞎猜；
  • AUC = 1.0：完美分类；
  • 业务经验：AUC > 0.8 一般算"强"，0.7–0.8 是"可用"，0.6–0.7 是"弱"，接近 0.5 意味着基本无效。
为什么金融场景偏爱 AUC 而不是 Accuracy？ 在类别不均衡（如信用违约、股价异常）场景，accuracy 会被多数类拖着走，只要预测全为多数类就能得到很高的 accuracy 却毫无信息量。AUC 是排序质量指标，只要模型能把正样本排在负样本之前就获得高分，与阈值选择无关，这对判断"信号是否有价值"至关重要。

三、Python 编程实现（一）癌症诊断二分类

3.1 数据概览

  • 数据集：sklearn.datasets.load_breast_cancer（UCI 威斯康星乳腺癌）
  • 样本数：569
  • 特征数：30 个数值特征（细胞核几何/纹理统计量：半径、周长、光滑度、凹陷度等）
  • 标签：良性(1) 357 例，恶性(0) 212 例
  • 划分：70% 训练 / 30% 测试，分层抽样，特征标准化（StandardScaler）

3.2 核心代码

# ① 加载数据
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.svm import SVC
from sklearn.metrics import confusion_matrix, roc_curve, auc

data = load_breast_cancer()

# ② 训练/测试集划分 + 标准化
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.3, stratify=data.target, random_state=42)
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test  = scaler.transform(X_test)

# ③ 构建并训练分类模型
models = {
    'LR':   LogisticRegression(max_iter=2000, random_state=42),
    'RF':   RandomForestClassifier(n_estimators=200, random_state=42),
    'SVM':  SVC(kernel='rbf', probability=True, random_state=42),
    'GBDT': GradientBoostingClassifier(random_state=42),
}
for name, clf in models.items():
    clf.fit(X_train, y_train)
    # ④ 评估 + 计算 AUC
    y_proba = clf.predict_proba(X_test)[:, 1]
    fpr, tpr, _ = roc_curve(y_test, y_proba)
    print(name, 'AUC =', round(auc(fpr, tpr), 4))

五、混淆矩阵 / ROC / AUC 深度解读

5.1 混淆矩阵在交易场景下的经济学含义

单元格意义交易场景
TP 真正例预测涨且真涨成功买入，盈利
FP 假正例预测涨但真跌错买了跌股，亏损
TN 真负例预测跌且真跌正确规避，未参与
FN 假负例预测跌但真涨错过机会，机会成本
  • 交易者最痛恨 FP（真金白银亏）→ 关注 Precision；提高信号阈值可提升 Precision，但会牺牲 Recall。
  • 医生最痛恨 FN（漏诊）→ 关注 Recall。领域不同，评价指标就要跟着换。

5.2 ROC 曲线的关键性质

  • ROC 曲线是"阈值扫描图"：对每一个阈值 t，把 proba ≥ t 视为正类，产生 (FPR, TPR)，连成曲线。
  • ROC 与阈值无关：改变阈值只改变在曲线上的工作点，不改变曲线本身。
  • ROC 与类别不均衡不敏感：即便正负比例 1:99，ROC 曲线依然可信；这一点 accuracy 不具备。

5.3 AUC 使用中的常见误区

  • AUC 高 ≠ 一定赚钱：交易还有滑点、手续费、容量、冲击成本。AUC 0.55 的模型如果换手过高，交易成本可能吃掉全部 alpha。
  • AUC 只衡量排序质量：不代表概率校准（predict_proba = 0.6 未必真有 60% 的正例概率）。
  • 样本外 AUC 才是真 AUC：训练集 AUC 高、测试集 AUC 低 = 过拟合，是量化开发中最常见的陷阱。
  • 不同任务的 AUC 不能直接比较：癌症的 AUC 0.99 与股价的 AUC 0.55 不能放在一起衡量"哪个模型更好"，任务难度天差地别。

6.2 机器学习在量化交易中的正确姿势

  1. 信号强度先于模型复杂度：AUC 0.52 换任何算法仍是 0.52，问题在特征而非模型。真正的 alpha 常来自另类数据（分析师情绪、供应链、订单流、卫星图像）。
  2. 严格时序切分：实盘必须走 WalkForward 或 PurgedKFold，避免"未来函数"泄漏。本报告随机分层切分只用于教学演示。
  3. 合理的评估指标：不仅看 AUC，还要看信号稳定性、换手率、扣费后收益、最大回撤等。分类精度只是万里长征第一步。
  4. 组合胜过单预测：即使单预测 AUC 仅 0.53，通过横截面选股（每天挑分数 top-N），叠加交易次数，组合 Sharpe 也能显著改善。
  5. 敬畏市场：类似癌症数据的高 AUC 在金融领域几乎不可能。看到宣称 AUC 0.95 的股票预测策略，先怀疑数据泄漏。
